## 二、后端新增检测指标

以下逻辑均在 `backend/pipeline/dispatch_loop.py`（或新拆出的 `metrics_calculator.py`）中实现，计算结果追加到 `stream_frame` JSON 消息，并在每分钟聚合时写入数据库。

---

### 2.1 占道率（Road Occupancy）

**计算公式**

```
occupancy = Σ(bbox_width_i × bbox_height_i) / ROI_area
```

- `ROI_area = (roi_x2 - roi_x1) × (roi_y2 - roi_y1)`，从 `device_settings` 读取，默认 `640×480`。
- 每帧实时计算，结果追加到 `stream_frame.detection.occupancy`（0~1 浮点）。
- 每分钟聚合：`avg_occupancy = mean(all_frame_occupancies)`，写入 `traffic_records`。

**LOS 服务水平等级（交通工程标准分类）**

| LOS | 占道率区间 | 含义 |
|-----|-----------|------|
| A   | 0% ~ 15%  | 畅通自由流 |
| B   | 15% ~ 35% | 稳定流 |
| C   | 35% ~ 55% | 临界稳定流 |
| D   | 55% ~ 75% | 不稳定流 |
| E   | 75% ~ 90% | 强制流（拥挤） |
| F   | > 90%     | 故障流（严重拥堵） |

每分钟写入 `traffic_records.los_grade`，可用于历史分析。

---

### 2.2 速度估算

**前提：摄像头标定**  
用户在设置页的"摄像头标定"功能中，在视频截图上点选两个已知真实距离的参考点（例如道路标线间距 3.5 m），系统计算并存储 `calibration_px_per_meter` 到 `device_settings`。未标定时速度显示为 N/A。

**计算逻辑（dispatch_loop 内）**

```python
# 内存维护 per-device 的 tracking_id → 最近5帧中心点列表
track_history: dict[int, deque]  # tracking_id → deque(maxlen=5)

def estimate_speed(track_id, cx, cy, fps, px_per_meter) -> float | None:
    history = track_history[track_id]
    if len(history) < 2:
        return None
    # 取最近5帧平均位移，稳定性优于相邻帧
    dx = cx - history[0][0]
    dy = cy - history[0][1]
    pixel_dist = (dx**2 + dy**2) ** 0.5
    elapsed_sec = (len(history) - 1) / fps
    speed_mps = (pixel_dist / px_per_meter) / elapsed_sec
    return speed_mps * 3.6  # 转换为 km/h
```

- 超速判断：`speed_kmh > device_settings.speed_limit_kmh` → 写入 `speed_events`，触发 `alert_type: speeding`（L3）。
- 每分钟聚合：`avg_speed_kmh`、`max_speed_kmh`、`speed_violation_count` 写入 `traffic_records`。
- `stream_frame.detection.avg_speed_kmh` 实时推送至前端。

### 2.3 逆行检测

**配置**：`device_settings.allowed_direction`（`up` / `down` / `both`）。

**检测逻辑**

```python
# dy > 0 表示向下（Y 坐标增大），dy < 0 表示向上
WRONG_WAY_FRAMES  = 5      # 连续帧数阈值
WRONG_WAY_MIN_PX  = 20     # 累计位移阈值（防抖）

def check_wrong_way(track_id, dy_history, allowed_direction) -> bool:
    if allowed_direction == 'both':
        return False
    recent = list(dy_history)[-WRONG_WAY_FRAMES:]
    if len(recent) < WRONG_WAY_FRAMES:
        return False
    cumulative = sum(recent)
    if allowed_direction == 'down' and cumulative < -WRONG_WAY_MIN_PX:
        return True  # 应向下但持续向上
    if allowed_direction == 'up' and cumulative > WRONG_WAY_MIN_PX:
        return True
    return False
```

触发条件满足时：`alert_type: wrong_way`，等级 L4。每分钟聚合 `wrong_way_count` 写入 `traffic_records`。

---

### 2.4 车头时距（Headway）

**实现**：`counter.py` 每次检测到车辆穿越计数线时，记录穿越时间戳。

```python
last_crossing_ts: dict[int, float]  # device_id → timestamp

def record_crossing(device_id: int) -> float | None:
    now = time.time()
    headway = None
    if device_id in last_crossing_ts:
        headway = now - last_crossing_ts[device_id]
    last_crossing_ts[device_id] = now
    return headway
```

- 内存维护最近10辆的车头时距列表，每次推送 `stream_frame.detection.avg_headway_sec`。
- 车头时距 < 1s → `alert_type: dense_flow`，等级 L2（信息预警）。
- 每分钟聚合 `avg_headway_sec`、`min_headway_sec` 写入 `traffic_records`。

### 2.5 排队长度检测

在现有异常停车检测（`parking_detector.py`）基础上扩展：

```python
def estimate_queue(parked_vehicles: list[Vehicle], px_per_meter: float) -> int:
    """
    将静止车辆按纵向坐标聚类（Y 差 < 50px 归一组），
    返回最大聚类的估算辆数。
    """
    if not parked_vehicles:
        return 0
    centers_y = sorted(v.cy for v in parked_vehicles)
    groups, current = [[centers_y[0]]], [centers_y[0]]
    for y in centers_y[1:]:
        if y - current[-1] < 50:
            current.append(y)
        else:
            groups.append(current := [y])
    max_group = max(groups, key=len)
    return len(max_group)
```

`queue_length > N`（可配置，默认 5）时触发 `alert_type: queue_detected`（L3），消息附排队辆数。

### 3.2 坐标映射

```typescript
const CANVAS_W = 320, CANVAS_H = 240;
const SCALE_X = CANVAS_W / 640;  // 0.5
const SCALE_Y = CANVAS_H / 480;  // 0.5

function toCanvas(px: number, py: number) {
  return { cx: px * SCALE_X, cy: py * SCALE_Y };
}
```

---

### 3.3 渲染循环

```typescript
// 每帧绘制前：用低透明度黑色覆盖，形成自然淡出拖尾
ctx.fillStyle = 'rgba(0, 0, 0, 0.15)';
ctx.fillRect(0, 0, CANVAS_W, CANVAS_H);

// 绘制辅助线
drawCountingLine(ctx, lineY * SCALE_Y);   // line_y 投影虚线
drawROIRect(ctx, roi);                     // ROI 半透明矩形框

// 绘制每辆车的轨迹和当前点
for (const [trackId, history] of trackHistoryMap) {
  drawTrail(ctx, history);     // 拖尾折线（渐变透明度）
  drawVehicleDot(ctx, vehicle, history.at(-1));
}
```

### 3.4 轨迹内存结构

```typescript
// useTrafficStore 中新增
trackHistoryMap: Map<number, Array<{ cx: number; cy: number; ts: number }>>

// 每帧 stream_frame 到达时更新，最多保留最近 30 帧
function updateTrackHistory(vehicles: Vehicle[]) {
  for (const v of vehicles) {
    const { cx, cy } = toCanvas(
      (v.bbox[0] + v.bbox[2]) / 2,
      (v.bbox[1] + v.bbox[3]) / 2
    );
    const hist = trackHistoryMap.get(v.tracking_id) ?? [];
    hist.push({ cx, cy, ts: Date.now() });
    if (hist.length > 30) hist.shift();
    trackHistoryMap.set(v.tracking_id, hist);
  }
  // 清理超过 3 秒未更新的 track
  for (const [id, hist] of trackHistoryMap) {
    if (Date.now() - hist.at(-1)!.ts > 3000) trackHistoryMap.delete(id);
  }
}
```

## 七、设置页升级

### 7.1 配置预览区（所见即所得）

左侧：实时视频缩略帧（1fps 低帧率，复用 WebSocket，降低流量），尺寸约 320×240。右侧：配置表单。

- 修改 `line_y` 时，预览图上的计数线实时移动（不需要保存即可预览）。
- 修改 `confidence` 时，预览图上显示当前阈值下的检测框数量变化提示。

---

### 7.2 ROI 拖拽配置

不再让用户手填 `roi_x1/y1/x2/y2` 四个数字。改为：

- 预览图上叠加可拖拽矩形框组件（纯 Canvas 实现）。
- 用户拖动四个角点调整 ROI，坐标自动反算到 640×480 空间。
- "保存"时将 `roi_x1/y1/x2/y2` 一并 PUT 到 `/api/devices/{id}/settings`。

---

### 7.3 摄像头标定功能

新增"速度标定"子区域：

1. 展示视频截图（静止帧）。
2. 用户在图上点击两个参考点（Canvas 点击事件）。
3. 弹出输入框：输入两点间真实距离（默认 3.5m，对应车道线间距）。
4. 系统计算 `px_per_meter = pixel_distance / real_distance`，保存到 `device_settings.calibration_px_per_meter`。
5. 标定完成后，设置页显示"已标定：X px/m"，速度相关功能自动启用。

---

### 7.4 逆行检测配置

新增"行驶方向"选择器（单选按钮组）：

- 不限制（双向）
- 仅允许向下行驶（Y 增大方向）
- 仅允许向上行驶（Y 减小方向）

选择后实时在预览图上叠加箭头提示，直观确认方向是否正确。
